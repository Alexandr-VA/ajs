## Библиотека для браузера и Node.js

### Легенда

Вы решили создать библиотеку, которую можно будет использовать как в браузере, так и в Node.js. Если вы помните, то мы рассматривали Moment.js, в котором был хитрый код.

Ранее для создания таких «универсальных» библиотек требовались сборщики вроде Webpack и транспайлеры вроде Babel. Однако современные среды — Node.js (начиная с версии 14) и все актуальные браузеры — нативно поддерживают ES-модули (ESM). Поэтому мы воспользуемся нативным подходом: никаких дополнительных инструментов сборки, только стандартные возможности языка и платформы.

Цель этого задания: убедиться, что один и тот же JavaScript-код (ES-модуль) работает в обеих средах — Node.js и браузере.

### Описание

#### Шаг 0. Создание репозитория

Перейдите на GitHub и создайте пустой репозиторий для вашего проекта (он нам понадобится для всех остальных действий). Мы будем подразумевать в рамках всей лабораторной работы, что вы назвали репозиторий `ajs`.

Склонируйте этот репозиторий на вашу локальную машину и все дальнейшие действия выполняйте в каталоге склонированного репо.

Сделайте commit и push, убедитесь, что код попал в ваш репозиторий на GitHub.

#### Шаг 2. Настройка package.json

Поскольку мы используем нативные ES-модули, дополнительные зависимости (webpack, babel и т.д.) нам **не нужны**. Достаточно правильно настроить `package.json`.

Добавьте (или измените) в вашем `package.json` следующие поля:

```json
{
  "type": "module",
  "exports": "./src/index.js",
  "files": [
    "src"
  ],
  "scripts": {
    "start": "node src/index.js"
  }
}
```

Где:
- `"type": "module"` — указывает Node.js интерпретировать `.js`-файлы как ES-модули (вместо CommonJS)
- `"exports"` — точка входа в пакет; используется современными версиями Node.js для разрешения импортов
- `"files"` — список файлов и каталогов, которые будут включены в пакет при публикации (только `src`)
- `"scripts"."start"` — скрипт для быстрой проверки библиотеки локально

**Важно**: поле `"main"` можно оставить без изменений (`src/index.js`) — оно будет использоваться старыми инструментами, не поддерживающими `"exports"`.

#### Шаг 3. Создание библиотеки

Создайте каталог `src`, а в нём файл `src/index.js` со следующим кодом:

```js
function info() {
  const m = new Map();
  m.set('name', 'ajs');
  m.set('version', '1.0.0');
  return m;
}

export default {
  info
};
```

Обратите внимание: это обычный ES-модуль. Никакой сборки не требуется — Node.js (14+) и все современные браузеры понимают `export`/`import` нативно.

Сделайте commit и push. Убедитесь, что файл попал на GitHub.

#### Шаг 4. Проверка в Node.js (npm link)

Для локальной проверки пакета без публикации удобно использовать `npm link`.

В каталоге вашего проекта с библиотекой (`ajs`) выполните:

```bash
npm link
```

Эта команда создаст глобальную символическую ссылку на ваш пакет.

Теперь создайте отдельный каталог для проверки:

```bash
mkdir ../ajs-test && cd ../ajs-test
npm init -y
```

Свяжите ваш пакет:

```bash
npm link @username/ajs
```

Не забудьте подставить свой username. Добавьте в `package.json` этого тестового проекта `"type": "module"`:

```json
{
  "type": "module",
  "scripts": {
    "start": "node src/index.js"
  }
}
```

Создайте файл `src/index.js`:

```js
import ajs from '@username/ajs';

console.log(ajs.info());
```

Запустите: `npm start`


#### Шаг 5. Проверка в браузере (http-server)

Теперь проверим, что тот же модуль работает в браузере. Для локального веб-сервера воспользуемся `npx http-server` — он запускается без установки.

Создайте в корне проекта файл `index.html`:

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>ajs browser test</title>
</head>
<body>
  <h1>ajs browser test</h1>
  <pre id="output"></pre>
  <script type="module">
    import ajs from './src/index.js';
    const data = Object.fromEntries(ajs.info());
    document.getElementById('output').textContent = JSON.stringify(data, null, 2);
    console.log(ajs.info());
  </script>
</body>
</html>
```

Обратите внимание: мы импортируем `./src/index.js` напрямую — браузер нативно поддерживает ES-модули через `<script type="module">`.

Запустите локальный сервер из корня проекта:

```bash
npx http-server -c-1 -o
```

Где:
- `-c-1` — отключает кеширование (удобно при разработке)
- `-o` — открывает браузер автоматически

В браузере откройте `http://localhost:8080/index.html`. Вы должны увидеть:

```json
{
  "name": "ajs",
  "version": "1.0.0"
}
```

Откройте DevTools (F12) → вкладка Console, чтобы убедиться, что `Map` также выводится в консоль.

**Почему нужен http-server, а не просто открыть файл?** ES-модули в браузере загружаются через сеть. При открытии файла через `file://` браузер блокирует импорты из соображений безопасности. Локальный сервер решает эту проблему.
